# Memory Types & Linking Spec

본 문서는 MemRosetta 시스템 내에서 "기억(Memory)"을 어떻게 정의하고, 어떤 유형으로 분류하며, 어떻게 연결할 것인지에 대한 명세입니다.

## 1. 기억의 정의 (What is a Memory?)

이 시스템에서 기억은 단순한 텍스트 덩어리(Chunk)가 아닙니다. 
기억은 **"독립적으로 이해 및 평가될 수 있는 최소 단위의 명제(Proposition) 또는 팩트(Fact)"**를 의미하며, 반드시 그 근거가 되는 **원천(Evidence/Artifact)**과 연결되어야 합니다.

## 2. 기억의 유형 (Memory Atom Categories)

인간의 인지 구조를 모방하여, 추출된 기억 원자(Atom)를 다음과 같은 6가지 핵심 범주로 분류합니다. 이 분류는 회상(Recall) 시 맥락적 가중치를 부여하는 기준이 됩니다.

1. **Episodic (에피소드 기억 / 사건)**
   * **정의:** 특정 시간, 장소, 맥락에서 발생한 구체적인 사건. "무엇이 있었나"
   * **예시:** "2026년 4월 1일, GPT-4를 이용해 로그인 버그를 수정함."
2. **Semantic (의미 기억 / 지식)**
   * **정의:** 맥락과 독립적인 일반적인 사실이나 개념. "무엇을 알게 됐나"
   * **예시:** "React `useEffect` 내에서 비동기 함수를 직접 반환하면 메모리 누수가 발생할 수 있음."
3. **Preference (선호도 / 성향)**
   * **정의:** 사용자의 명시적/암시적 호불호 및 설정. "무엇을 선호하나"
   * **예시:** "사용자는 탭(Tab) 대신 2개의 스페이스(Space) 들여쓰기를 선호함."
4. **Decision (결정 및 근거)**
   * **정의:** 과거에 내린 기술적/일상적 판단과 그 합리적 근거. "무엇을 결정했나"
   * **예시:** "RDBMS로 MySQL 대신 PostgreSQL을 선택함. (이유: JSONB 지원 및 확장성 향상)"
5. **Procedural (절차적 기억 / 방법론)**
   * **정의:** 특정 작업을 수행하는 순서, 워크플로우, 스니펫. "어떻게 하는가"
   * **예시:** "새로운 엔드포인트 추가 시: 1) 라우터 정의, 2) Zod 스키마 작성, 3) 컨트롤러 구현 순으로 진행함."
6. **Identity (자아 및 정체성)**
   * **정의:** 시스템의 주인(User)에 대한 핵심적인 특징이나 장기적 목표. "나는 어떤 사람인가"
   * **예시:** "나는 시스템 아키텍처를 설계할 때 단기 속도보다 장기 유지보수성을 중시하는 개발자임."

## 3. 신경망 연결 계층 (Linking Layer & Edge Types)

개별 기억 원자들은 독립적으로 존재하지 않고 지식 그래프를 형성해야 합니다. MemRosetta는 기억 간의 관계(Edge)를 생성하여 연상적 회상(Associative Recall)을 가능하게 합니다.

### 기본 관계 유형 (Core Link Types)
* `related`: 의미적으로 연관되어 있음 (양방향/단방향).
* `updates`: 기존 기억을 새로운 기억이 갱신함 (버전 관리 및 과거 기억 감가상각에 사용).
* `contradicts`: 두 기억이 서로 모순됨 (추후 에이전트의 확인/병합 프로세스 트리거).
* `supports`: 특정 결정(Decision)이나 사실(Semantic)을 뒷받침하는 근거.
* `derived_from`: A라는 기억이 B라는 기억(또는 Artifact)으로부터 도출/요약됨.

### 확장 관계 유형 (Future Scope)
* `habitual`: 반복적으로 관찰되는 패턴.
* `goal_related`: 특정 장기 목표(Identity) 달성과 연관됨.
* `people_related`: 특정 인물/동료와의 상호작용과 연관됨.

## 4. Source-First 파이프라인 (Ingestion Lifecycle)

데이터가 시스템에 들어와 기억으로 정착되는 과정은 다음과 같습니다.
1. **Raw Artifact 보존:** 원본 데이터(코드, 채팅, 문서)를 불변 상태로 저장.
2. **Propositionize:** LLM을 활용해 Artifact에서 독립된 명제(Atom)들을 추출.
3. **Typing & Dedup:** 추출된 명제를 6가지 타입으로 분류하고, 기존 기억과 중복(Dedup)을 검사.
4. **Link & Evidence:** 기존 기억과의 관계(Edge)를 생성하고, 원본 Artifact로의 출처(Evidence)를 연결.
